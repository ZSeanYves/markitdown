# phase-1.5：鲁棒性巩固计划（基于当前 markitdown-mb 状态）

> 目标：在不打断当前稳定结构（`bytes -> ZipArchive -> OoxmlPackage -> format parser`）的前提下，优先补齐边界样本、负例样本、已知限制覆盖与少量底层收尾。

## 0) 当前基线（用于排序依据）

- OOXML 迁移已完成，DOCX/XLSX/PPTX 均已在共享底座上运行。
- PDF 仍是主要技术债，尤其依赖外部提取与启发式。
- 现有样本规模（输入样本数）大致为：
  - PDF: 12
  - PPTX: 46
  - XLSX: 12
  - DOCX: 13

由此可知：
- **PPTX 样本量已不小，适合做“结构误判负例补强”**。
- **PDF 风险最高但样本相对少，最应优先补“失败模式覆盖”**。
- **XLSX/DOCX 基本稳定，适合补小而关键的边界样本**。

---

## 1) 最值得优先补的测试样本（按优先级）

## P0（立即补）

### A. PDF：失败模式与已知限制样本（最高优先）

1. **双栏/伪双栏阅读顺序负例**
   - 目标：验证“不会把左右栏错误拼接成一段”。
   - 预期：宁可分段保守，也不要跨栏合并。

2. **脚注/页眉页脚干扰样本（变体）**
   - 目标：覆盖页码、页脚注释、章节名重复但不完全一致场景。
   - 预期：重复噪声尽量去掉；正文不误删。

3. **标题误判负例（短句、全大写、带编号）**
   - 目标：减少 heading false positive。
   - 预期：短句与普通强调行不应轻易升格为标题。

4. **跨页段落“应断不断/应连不连”对照组**
   - 目标：覆盖两类边界：
     - 下一页首行是新节（不应 merge）
     - 下一页首行是续写（应 merge）

5. **OCR fallback 可回归样本（小规模）**
   - 目标：至少 2–3 个扫描件质量等级（清晰/中等/差）基线。
   - 预期：允许质量一般，但输出结构要“可解释可复现”。

### B. PPTX：负例优先于新能力

1. **table-like 误判负例簇（继续加）**
   - 目标：文本卡片墙、时间线、关键词云、图注散点等“不该变表格”的布局。
   - 预期：保持段落/列表，不误转 Markdown table。

2. **reading-order 冲突样本**
   - 目标：Z-order 与坐标顺序冲突、旁注/角标干扰。
   - 预期：主阅读流稳定，旁注不抢主干。

3. **组对象 + 多段落标题边界样本**
   - 目标：分组后标题、正文、注释混排时不串行。

## P1（随后补）

### C. XLSX：现实工作簿边界

1. **merged cells + sparse 区域混合**
2. **日期/时间格式边界（1900/1904、仅时间、跨日时间）**
3. **公式缓存值与类型混合（string/number/error）**
4. **多 sheet 大小差异极大（空表 + 超稀疏表）**

### D. DOCX：语义样式边界

1. **blockquote 真样本（README 已明确缺口）**
2. **list restart / start override / 多级编号切换**
3. **代码样式误判负例（普通缩进段不应进 code block）**
4. **表格内复杂段落（换行+列表）**

## 2) phase-1.5 可做的“底层收尾”（不打断稳定结构）

以下建议都遵循“只加护栏，不重开架构”：

## A. ZIP / OOXML 层（小步增强）

1. **错误分类与可观测性增强**
   - 为 Zip/Ooxml 常见失败点补更细错误码/错误消息（缺 part、rels 断链、content type 缺失）。
   - 收益：回归失败时定位更快，不改主流程。

2. **路径正规化一致性检查**
   - 在 part path / rel target resolution 上补统一 normalize（`./`、`../`、重复斜杠）。
   - 收益：减少边缘包体导致的偶发失败。

3. **“能力边界”显式测试**
   - 针对 README 已声明“不支持 ZIP64/加密/多盘”等，补“明确失败”的测试（negative expected fail）。
   - 收益：把“已知不支持”从口头约定变成可回归契约。

## B. PDF 层（不改主策略，仅稳态护栏）

1. **提取器选择打分日志（可选开关）**
   - 输出每个候选提取结果的关键评分因子。
   - 收益：误判时可快速复盘，不改变默认行为。

2. **页噪/标题启发式阈值外置成小配置常量**
   - 先做“集中定义”，不做大规模参数化系统。
   - 收益：后续调参更安全、diff 更可控。

3. **OCR 路径失败信息分层**
   - 区分“无文本层 + 未安装工具”“工具执行失败”“OCR结果为空”。
   - 收益：减少问题排查成本。

## C. PPTX/XLSX/DOCX 层（解析稳态）

1. **解析阶段的空节点/坏节点容错早返回**
2. **关键启发式旁路条件统一（避免一处修复引发另一处回归）**
3. **对外行为不变的重构只限函数内聚/命名清理**

---

## 3) 执行顺序建议：先从哪块开始

建议顺序：

1. **先 PDF**（风险最高 + 样本相对不足 + README 明确技术债）
2. **再 PPTX**（功能面广、启发式复杂，负例最能稳住）
3. **再 DOCX**（补 README 已知缺口：blockquote 真样本）
4. **最后 XLSX**（当前稳定，做边界补齐即可）

可操作的两周节奏（示例）：

- **Week 1**
  - PDF: 4 个文本负例 + 2 个 OCR 分级样本
  - 同步补 1 个 extractor 评分可观测性开关

- **Week 2**
  - PPTX: 6–8 个误判负例（以 table-like/read-order 为主）
  - DOCX: 2–3 个样式边界样本（含 blockquote 真样本）
  - XLSX: 2–3 个日期/稀疏边界样本

---

## 4) 本阶段验收标准（建议）

1. 新增样本中，**负例占比 ≥ 50%**。
2. 每个格式至少覆盖 1 条“已知限制”并以测试固化。
3. `samples/diff.sh` 与 `moon test` 全通过。
4. 若行为变更，必须有对应样本与预期更新，不接受“无样本改启发式”。

---

## 5) 一句话结论

- phase-1.5 的核心不是“再造架构”，而是把你现在已经稳定的底座变成“**可解释地稳定**”：
  - **PDF 先补失败模式**，
  - **PPTX 补误判负例**，
  - **DOCX/XLSX 补已知边界**，
  - 仅做不扰动主流程的底层收尾。
